# Internationalization (i18n) Rules

## Setup

- Uses **i18next**.
- Locale: `SFDX_HARDIS_LANG` env var (default: `en`).
- Supported: `de`, `en`, `es`, `fr`, `it`, `ja`, `nl`, `pl`, `pt-BR`.
- Translation files: `src/i18n/<locale>.json` -- flat JSON, camelCase keys, alphabetically sorted.

## Two Message Systems

1. **i18n (`src/i18n/*.json`)**: Runtime translations via `t()` -- primary for user-visible strings.
2. **Oclif messages (`messages/*.md`)**: SF CLI framework messages via `Messages.importMessagesDirectoryFromMetaUrl()` -- for flag descriptions and summaries.

## Key Naming Conventions

- Use `camelCase`, starting with a lowercase letter.
- Key should be a compressed English summary of the message:
  - `"No apex logs to delete."` -> `noApexLogsToDelete`
  - `"Processing file {{file}}..."` -> `processingFile`
  - `"Error while deploying metadata: {{message}}"` -> `errorWhileDeployingMetadata`
- Keys must be unique across the whole file.
- Reuse existing keys when possible.

## What Must Use `t()`

- `uxLog()` calls with user-visible messages.
- `prompts()` -- `message`, `description`, `placeholder`, `choices[].title` properties.
- `sendReportFileMessage()` -- second argument (message).
- `WebSocketClient.sendProgressStartMessage()` -- first argument.

## Exceptions (no `t()` needed)

- Debug stack traces.
- URLs only.
- `uxLog("other", ...)` calls.
- Pure variable/expression strings.

## When Adding a Key

- Add to **all 9 locale files** simultaneously (`en`, `de`, `es`, `fr`, `it`, `ja`, `nl`, `pl`, `pt-BR`).
- Keep JSON files sorted alphabetically by key.
- Keep `{{varName}}` interpolation placeholders exactly as-is.
- Keep `\n`, `<br/>`, emoji, and markdown formatting intact.
- Do NOT translate `[]` markers (e.g. `[sfdx-hardis]`, `[SKIP]`).
- Do NOT translate technical terms: merge, commit, branch, sandbox, scratch org, package.xml, Apex, SOQL, LWC, CLI flags, env var names.
- Keep brand names untranslated: Salesforce, SFDMU, Git, GitHub, GitLab, JIRA, VS Code, Cloudity, sfdx-hardis, Azure DevOps, Docker, Bitbucket, Cloudflare, ServiceNow, MermaidJS.
- "org" stays as "org" in all languages.
